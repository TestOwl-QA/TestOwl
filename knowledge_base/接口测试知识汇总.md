## 一、接口测试概述

接口测试是确保游戏模块间通信可靠性和功能完整性的关键环节，目标是验证游戏服务端与客户端之间的数据交互、逻辑处理以及业务流程是否按照设计规范正常运行。

### 1.1 游戏接口测试范围

| 测试类型 | 说明 |
|---------|------|
| 单接口功能验证 | 验证单个接口的输入输出 |
| 业务流程测试 | 多接口串联的业务场景 |
| 性能测试 | 高并发、响应时间 |
| 安全测试 | 认证、权限、加密 |

### 1.2 游戏典型接口

- 登录/注册接口
- 道具购买接口
- 任务系统接口
- 排行榜接口
- 角色操作接口
- 支付/充值接口

## 二、接口测试用例设计方法

### 2.1 用例设计核心原则

```
接口测试 = 输入参数 + 预期输出 + 校验规则
```

**设计方法**：

| 方法 | 说明 | 适用场景 |
|-----|------|---------|
| 等价类划分 | 将输入分为有效/无效等价类 | 减少冗余用例 |
| 边界值分析 | 测试边界情况 | 数值范围、长度限制 |
| 因果图/判定表 | 逻辑复杂的多条件组合 | 任务完成条件、成就解锁 |
| 场景法 | 模拟真实用户操作流程 | 业务流程测试 |

### 2.2 输入参数测试用例设计

**必填参数校验**：
- 区分必填参数和可选参数
- 每条用例覆盖所有必填参数
- 可选参数进行组合测试

**数据类型验证**：
| 参数类型 | 测试要点 |
|---------|---------|
| 数值型 | 0、负数、小数、边界值、超限值 |
| 字符串 | 空值、超长、特殊字符、SQL注入 |
| 列表/字典 | 元素数量、顺序、重复、矛盾元素 |

**边界值测试**：
```
测试范围：最小值、最大值、边界值-1、边界值+1
示例：分页参数page_size测试 0, 1, 50, 51（上限50）
```

### 2.3 处理逻辑测试用例设计

| 测试类型 | 说明 | 示例 |
|---------|------|------|
| 大数据量测试 | 数据量大时接口表现 | 批量分解99999个道具 |
| 幂等性测试 | 重复请求不产生副作用 | 重复领取奖励只生效一次 |
| 环境异常测试 | 环境异常时接口响应 | 非活动时间访问活动接口 |
| 并发测试 | 多用户同时调用 | 压力测试场景单独设计 |
| 安全测试 | 恶意攻击防护 | 非法ID注入攻击 |

## 三、接口测试用例类型

### 3.1 功能测试用例

验证接口在正常输入下能否正确处理

**示例：登录接口**

| 用例ID | 测试场景 | 输入数据 | 预期结果 |
|--------|---------|---------|---------|
| TC001 | 正常登录 | 正确用户名密码 | 返回200，Token非空 |
| TC002 | 密码错误 | 正确用户名、错误密码 | 返回401，错误提示 |
| TC003 | 账号不存在 | 不存在的用户名 | 返回404，账号不存在 |
| TC004 | 参数缺失 | 缺少密码字段 | 返回400，密码不能为空 |

### 3.2 边界测试用例

验证接口在边界条件下的表现

**示例：角色等级上限**

| 用例ID | 测试场景 | 输入数据 | 预期结果 |
|--------|---------|---------|---------|
| TC001 | 正常升级 | 经验值足够 | 等级+1，属性正确更新 |
| TC002 | 达到上限 | 经验值达到最高等级 | 返回已达最高等级提示 |
| TC003 | 超过上限 | 经验值超过上限 | 等级不变，经验值溢出处理 |

### 3.3 异常测试用例

验证接口对异常输入的处理能力

**示例：认证异常**

| 用例ID | 测试场景 | 输入数据 | 预期结果 |
|--------|---------|---------|---------|
| TC001 | Token为空 | 空Token | 返回401未授权 |
| TC002 | Token过期 | 过期Token | 返回401，需重新登录 |
| TC003 | Token篡改 | 非法Token | 返回401，拒绝访问 |

### 3.4 权限测试用例

验证不同权限用户的接口访问

**示例：越权访问**

| 用例ID | 测试场景 | 操作 | 预期结果 |
|--------|---------|------|---------|
| TC001 | 普通用户访问管理员接口 | 调用管理接口 | 返回403权限不足 |
| TC002 | 用户A访问用户B数据 | 传入B的ID | 返回403，拒绝访问 |

### 3.5 幂等性测试用例

验证重复请求不会产生副作用

**示例：重复领取奖励**

```python
def test_idempotent_claim_reward():
    """测试重复领取奖励"""
    # 第一次请求
    response1 = claim_reward(reward_id=1001)
    assert response1.status_code == 200
    
    # 第二次请求
    response2 = claim_reward(reward_id=1001)
    assert response2.status_code == 400
    assert response2.json()['error'] == '奖励已领取'
```

### 3.6 安全测试用例

**SQL注入测试**：
```python
def test_sql_injection():
    """测试SQL注入攻击"""
    malicious_input = "' OR '1'='1"
    response = create_character(malicious_input, 1)
    assert response.status_code == 400
    assert response.json()['error'] == '非法输入'
```

**XSS攻击测试**：
```python
def test_xss_attack():
    """测试XSS攻击"""
    malicious_input = "<script>alert('xss')</script>"
    response = update_username(malicious_input)
    assert response.status_code == 400
```

## 四、接口测试核心要素

### 4.1 用例结构

| 要素 | 说明 | 示例 |
|-----|------|------|
| 用例ID | 唯一标识 | TC_LOGIN_001 |
| 接口信息 | URL和请求方法 | POST /api/login |
| 请求参数 | Headers、Body | {"username":"test"} |
| 预期结果 | 状态码、响应数据 | 200, {"token":"xxx"} |

| 优先级 | 用例重要程度 | P0（最高） |

### 4.2 响应验证要点

| 验证项 | 说明 |
|-------|------|
| 状态码 | 200/201/400/401/403/404/500 |
| 响应头 | Content-Type、Set-Cookie等 |
| 响应体 | 字段存在性、数据类型、值范围 |
| 响应时间 | 关键接口<500ms |
| 数据库验证 | 接口返回与数据库一致 |

### 5.2 Python接口测试示例

**基础请求示例**：
```python
import requests

def test_game_login():
    """游戏登录接口测试"""
    url = "https://api.game.com/v1/login"
    payload = {
        "username": "testuser",
        "password": "password123"
    }
    
    response = requests.post(url, json=payload)
    
    # 验证状态码
    assert response.status_code == 200
    
    # 验证响应结构
    data = response.json()
    assert "token" in data
    assert data["status"] == "success"
    
    # 验证响应时间
    assert response.elapsed.total_seconds() < 1.0
```

**参数化测试**：

## 八、接口测试指标与评估标准

### 8.1 功能指标

| 指标 | 说明 | 要求 |
|-----|------|------|
| 接口响应状态码 | 正确返回HTTP状态码 | 符合API规范 |
| 错误码与提示 | 异常输入返回正确错误码 | 错误信息清晰准确 |
| 数据一致性 | 接口返回与数据库一致 | 100%一致 |

### 8.2 性能指标

| 指标 | 说明 | 目标值 |
|-----|------|--------|
| 响应时间 | 接口响应延迟 | 关键接口<100ms |
| 吞吐量 | 每秒处理请求数 | 根据业务规模 |
| 并发数 | 同时处理请求数 | 根据业务规模 |
| 成功率 | 请求成功比例 | ≥99.9% |

### 8.3 安全指标

| 指标 | 说明 | 要求 |
|-----|------|------|
| 身份验证 | Token/OAuth机制 | 必须验证 |
| 权限控制 | 越权访问防护 | 必须防护 |
| SQL注入 | 恶意SQL过滤 | 必须过滤 |
| 数据加密 | 敏感数据传输 | 必须加密 |

## 九、接口测试常见缺陷

### 9.1 参数处理缺陷

- 未校验参数边界导致缓冲区溢出
- 参数类型校验不严格
- 必填参数缺失未处理

### 9.2 权限漏洞

- 垂直越权：普通用户访问管理员接口
- 水平越权：用户A访问用户B数据

### 9.3 数据问题

- 业务状态机转换异常
- 响应中包含敏感信息
- 数据库与接口返回不一致

### 9.4 并发问题

- 并发场景下数据覆盖
- 重复处理同一请求
- 资源竞争条件
