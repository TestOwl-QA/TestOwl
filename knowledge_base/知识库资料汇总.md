## 一、数值测试方向

### 1.1 核心测试维度（来源：CSDN、51Testing）

**基础数值校验**
- 角色属性验证（生命值、攻击力、防御力）
- 数值表格与实际运行数据一致性对比
- 技能效果验证：伤害计算公式、冷却时间、范围影响

**经济系统评估**
- 货币流通机制检查
- 道具掉落率验证
- 商店价格合理性
- 案例：某MMORPG发现任务奖励过高导致货币贬值

**战斗平衡性检测**
- PVP/PVE对抗模拟
- 不同职业能力差异分析
- 胜率统计与调整

**边界条件测试**
- 玩家等级最大值（如99级能否升到100级）
- 背包最大容量
- 金币最大值
- 装备满级增益叠加

### 1.2 伤害公式示例（来源：CSDN文库）

伤害减免公式：
```
伤害减免百分比 = 防御力 / (防御力 + K1 × 人物等级 + K2)
```

测试方法：
- 输入策划表格参数，验证输出是否匹配
- 检查防御力为0、负数、极大值时的行为
- 验证等级提升是否影响防御收益

### 1.3 测试流程（来源：CSDN文库）

1. 需求评审阶段：了解功能模块和算法模型
2. 编写专项用例：基于等价类划分和边界值分析
3. 执行测试：手动探索 + 自动化回归
4. 记录异常：截屏保存，提交缺陷管理系统
5. 回归复查：验证修复后不再复现

### 1.4 工具推荐（来源：CSDN博客）

| 工具 | 用途 |
|------|------|
| Excel/Pandas | 数值公式验证 |

| Unreal Automation | Unreal引擎自动化测试 |

## 二、关卡测试方向

### 2.1 地图边界测试（来源：51Testing）

测试内容：
- 地图边缘是否有空气墙
- 悬崖边缘是否卡模型
- 玩家能否走到地图外
- 副本入口权限检查

测试方法：
- "极限探索"法：往地图四角、悬崖边缘走
- "数据边界"法：测试数值极限
- "场景联动"法：跨场景状态检查

### 2.2 关卡连通性验证（来源：原创力文档）

```python
import networkx as nx

def validate_level_connectivity(level_data):
    # 创建无向图
    G = nx.Graph()
    # 将可通行点作为节点
    for i in range(len(level_data)):
        for j in range(len(level_data[0])):
            if level_data[i][j] == 0:  # 可通行
                G.add_node((i, j))
    # 添加相邻边
    # ... 省略边添加代码
    return nx.is_connected(G)
```

### 2.3 自动化测试类型（来源：Unreal官方文档）

| 测试类型 | 说明 |
|---------|------|
| 单元测试 | API级别验证 |
| 功能测试 | 系统层面测试（PIE、分辨率等） |
| 冒烟测试 | 1秒内完成的快速测试 |
| 内容压力测试 | 加载所有资源，检测崩溃 |
| 截屏比较 | 不同版本渲染差异对比 |

## 四、真实案例分析（来源：各文档）

### 案例1：MOBA英雄强度调整
- 初期属性适中
- 胜率超平均则下调伤害
- 太弱则提高属性
- 目标：维持所有英雄胜率在合理区间

### 案例2：某MMORPG经济系统问题
- 问题：技能冷却设置不合理，经济通胀严重
- 原因：数值未经过系统测试直接上线
- 解决：建立数值测试体系，上线前100%验证

### 案例3：原神边界测试
- 用钟离岩脊技能在地图边缘搭楼梯
- 检测能否爬出地图
- 发现漏洞需添加空气墙
