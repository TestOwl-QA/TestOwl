# TestOwl 上下文理解问题修复

## 问题描述
AI回复与问题不相关，无法记住对话历史（上下文理解缺失）

## 根本原因
1. **前端未传递history**: 前端发送消息时没有包含对话历史
2. **前端未保存历史**: 前端没有维护对话历史数组
3. **后端prompt问题**: 系统prompt没有强调上下文记忆的重要性

## 修复内容

### 1. 前端修复 (web/index.html)

#### 1.1 添加历史记录数组
```javascript
let chatHistory = [];  // 对话历史记录
```

#### 1.2 修改addMessage函数
- 添加消息时保存到chatHistory
- 限制历史记录长度为最近40条（20轮对话）

#### 1.3 修改sendMessage函数
- 发送请求时包含history字段

#### 1.4 修改typeWriter函数
- AI回复完成后保存到chatHistory

### 2. 后端修复 (web/chat_handler.py)

#### 2.1 优化普通对话prompt
- 明确指示AI记住对话历史
- 强调回答要与上下文相关
- 增加历史记录保留数量（3条→10条）

## 修复后的代码变更

### web/index.html
```javascript
// 新增: 对话历史数组
let chatHistory = [];

// 修改: addMessage函数保存历史
chatHistory.push({role: role, content: content});
if (chatHistory.length > 40) {
    chatHistory = chatHistory.slice(-40);
}

// 修改: sendMessage传递历史
body: JSON.stringify({
    message: message,
    session_token: token,
    file_id: currentFileId,
    history: chatHistory  // 新增
})

// 修改: typeWriter保存AI回复
chatHistory.push({role: 'assistant', content: text});
```

### web/chat_handler.py
```python
# 修改: 优化prompt，强调上下文记忆
context_messages = []
for m in history[-10:]:  # 从3条增加到10条
    role = m.get('role', '')
    content = m.get('content', '')
    if role and content:
        context_messages.append(f"{'用户' if role == 'user' else 'AI'}: {content}")

prompt = f"""你是TestOwl测试助手...

重要：你必须记住对话历史，回答要与上下文相关。

对话历史:
{context}

当前用户问题: {user_msg}

请直接回答用户问题，保持简洁专业。如果用户问的是之前提到过的内容，请基于历史对话回答。"""
```
